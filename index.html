Just a few patches to make the ruby libvirt API a bit less crap, with a view to allowing someone to write simple snapshot-based backup scripts.

Makes the following possible:

Libvirt::open.domains.find_all{|a|a.active?}.each do |d|
  # Libvirt::Connection#domains is a list of domain objects, instead of IDs
  d.create_snapshot do |s|
    # With a block, creates snapshot and deletes it when the block is done
    puts s.backing_store # details about each disk and where it's stored

    # Takes a backup of a snapshot of every disk of this domain
    s.qemu_img_backup_commands("/var/backups/").each do{|s|system s}
    # Backups will be (e.g) /var/backups/webvm-hda.qcow
  end
end

You can use fpm (from the Makefile) to create debian or CentOS packages with `make'. You need rubygems for fpm, but not to use the final package.

On CentOS, you'll need the EPEL repo to install the final package, and rpmbuild to be able to create the RPMs with FPM.
